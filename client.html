Dorkemon

This is my attempt to create a browser based game for the Nintendo 3DS

It's going to be massive sandbox game in the 2d top-down style of old RPGs like Pokemon or Zelda. You can load images from the internet, crop and resize them into tiles, then paste them all over an ever expanding canvas. You can build a house, or a garden, maybe even recreate a level from your favorite old 2D RPG. As you walk around you can meet other players, and create your own custom avatar with various animations.

Dev Notes

Aug 30th, 2013

I started this project using a Node tool called Harp to preprocess my html css and javascript. I use Sass at work and have played with Jade as well and there's something nice about their simple syntax. Since Harp can also process CoffeScript I decided to give it a shot as well. Well that was a mistake. CoffeeScript did nothing but trip me up at every turn. Then I started using Knockout to manage the bottom screen interface and realized that Jade doesn't support multiline attributes which makes my data-binds hard to read. Then I realized that if I'm using Knockout to generate dynamic elements, I might as well just inline the styles for those elements. So I yanked all the generated code from their disparate files and put them in a single HTML file.

I know a lot of people will say this is a bad idea. I'll admit a single file application would be problematic for several developers to work on at once. But this is just me, and I don't like having to keep track of what code is in what file.
